Linux Commands

cd [directory]

  • changes the current directory
    • if no directory name is entered, it will change to the user's home directory
  • example of usage: a user wanting to change from their home directory to their directory titled projects would use "cd projects"

ls [directory]

  • lists the contents of the directory
    • if no directory name is entered, it will list the contents of the current directory that the user is in
  • example of usage: a user wants to delete a directory but decides to check to see if the directory is empty first; they would use "ls [directory]" to check

mkdir [directory]

  • creates a new directory
    • note: [directory] parameter refers to the name of the new directory that the user desires to create
  • example of usage: a user wanting to create a new directory entitled "CSP" would use the command "mkdir CSP"

cat [file]

  • used to display a file
    • note: [file] parameter refers to the name of the new file that the user wishes to display
  • example of usage: a user wanting to display a variable they previously initiated would use "cat [variable name]"

nano [file]

  • used to edit an existing file or create a new file if one does not already exist under the specified name
    • note: [file] parameter refers to the name of the file that the user desires to modify/create
  • example of usage: a user wanting to edit a file named rpistatus would use "nano rpistatus"

man [command]

  • displays the properties of the specified raspberry pi commands
    • note: [command] parameter refers to the name of the command that the user wants more information about
  • example of usage: a user wanting a description of the nano command would type "man nano"

cp [file] [directory to copy into]

  • used to copy an already existing file
    • note: [file] parameter refers to the name of the file that the user desires to copy
  • example of usage: a user wanting to copy the file "pistatus.cgi" to the directory "/usr/lib/cgi-bin/student3 would type "pistatus.cgi /usr/lib/cgi-bin/student3"

rm [file]

  • used to remove an existing file
    • note: [file] parameter refers to the name of the file that the us$
    • note: to remove a directory, use fmdir [directory]
  • example of usage: a user wanting to remove the file "peanut butter" would type "rm peanut butter"

chmod +x [file]

  • used to edit to make a file accessible
    • note: [file] parameter refers to the name of the file that the us$
  • example of usage: a user wanting to edit the accessibility of the file "jelly" would type "chmod +x jelly"

git push

  • used to push new changes from a local repository to the public repository
    • note:"git push" command usually preceded by "git add [file]" to save changes from a working directory to a temporary stage area and then "git commit -m 'message'" to move changes from a temporary staging area to the local repository under the edit title 'message'
  • example of usage: a user wanting to save new coding in the file "linux.html" to the public website would type: "git add linux.html" then "git commit -m linux1" and finally "git push"
